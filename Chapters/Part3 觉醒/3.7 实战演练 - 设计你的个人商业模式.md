# 3.7 实战演练：设计你的个人商业模式

> [!IMPORTANT]
> **本章寄语**：知道 100 个商业故事，不如亲手规划一次自己的生意。在这一章里，我们将把前面学到的“一人公司意识”、“个人品牌积淀”、“时间资产分配”和“商业核心算账”等理论融为一炉，**亲手绘制出属于你自己的“个人商业模式画布”**。完成后，你将清晰地看清你当前的核心资产在哪、客户在哪、以及如何赚到你的第一笔复利金。

学完了本章的前六节，你已经完成了商业思维的“扫盲”。你明白自己就是一家公司，也知道了不能陷入“自嗨陷阱”，甚至知道了怎么在校园的“沙盒环境”中低成本做实验。

现在，是时候把这些零散的认知，拼装成一张具有高度执行力的**商业战术地图**了。

我们将使用享誉全球的**“个人商业模式画布”（Personal Business Model Canvas）**。它将复杂的企业运营，浓缩在九个紧密关联的格子里。当你填满这九个格子，你的“一人公司（OPC）”运作链路就会变得一目了然。

## 二、 模块详解与填色指南

请对照以下说明，在大脑中或纸上开始勾勒你的格子：

1.  **客户细分（我帮谁？）**：谁会为你的能力买单？越具体越好。
    *   *错误定义*：“所有需要英语的人。”
    *   *正确定义*：“大四需要快速翻译并抛光学术论文摘要的工科学生。”
2.  **价值主张（我怎么帮？）**：你解决了他们什么痛点？
    *   *示例*：“提供 10 分钟出稿、地道学术风、带有一对一逻辑解析的英文摘要抛光服务，让其能通过导师的一审。”
3.  **渠道通路（他们怎么知道我？）**：你通过什么路径触达他们？
    *   *示例*：“小红书学术笔记引流、学校表白墙发帖、宿舍朋友圈口碑推荐。”
4.  **客户关系（我如何互动？）**：你跟客户是一锤子买卖，还是长期陪伴？
    *   *示例*：“建立‘论文互助微信群’，每周分享 AI 伴读学术前沿，建立高粘性信任。”
5.  **核心资源（我是谁，我有什么？）**：你手中握着的“核武器”。
    *   *示例*：“我自己编写的 `抛光大师.md` 提示词 Skill；熟练使用 Trae 的前端编程能力；对学术论文格式的理解。”
6.  **关键业务（我做什么？）**：你每天在把时间投在什么地方？
    *   *示例*：“运行 n8n 工作流自动处理排版；审核 AI 输出的英文语感并做微调。”
7.  **重要合作（谁帮我？）**：支撑你的外部杠杆。
    *   *示例*：“DeepSeek 提供廉价 API 支持；深圳市龙岗区提供的 OPC 虚拟注册地址；GitHub 托管代码。”
8.  **成本结构（我付出什么？）**：你每个月的硬开销。
    *   *示例*：“大模型 API 费用（预计 30 元/月）、表白墙广告费（50 元/月）、我的时间投入。”
9.  **收入来源（我得到什么？）**：你怎么收钱？
    *   *示例*：“单篇摘要抛光 19 元/次，包年订阅 199 元/年，微信收款或 Lemon Squeezy 收单。”

## 三、 案例演练：大二学生小华的“一人公司”画布

小华利用在第二章学到的 `Trae 编程` 和 `n8n 自动化`，为大一新生设计了一个“校园生活助手兼日程自动提醒小工具”。以下是他的画布形态：

### 小华的个人商业模式画布 (Case Study)

| 模块名称 | 小华的设计细节 | 运作逻辑与杠杆 |
| :--- | :--- | :--- |
| **1. 客户细分** | 本校大一、大二课表极度复杂、经常忘记抢课和点名的学生。 | 锁定高频遗忘、有时间焦虑的年轻群体。 |
| **2. 价值主张** | 绑定微信的“智能小课表”，每天早晨自动发微信通知当天课程、抢课提醒并附带天气。 | 解决了“频繁打开选课系统看课表”的厌烦感。 |
| **3. 渠道通路** | 学校新生 QQ 群群发展示视频、食堂门口海报、舍友之间自发推荐。 | 校园强社交链条冷启动。 |
| **4. 客户关系** | 微信小管家式服务。自动回复常见抢课攻略。 | 极具温度的校园伴侣形象。 |
| **5. 关键业务** | 调试课表导入算法，确保不同院系课表 100% 正确；维护微信 Webhook 消息发送。 | 确保交付质量无 BUG。 |
| **6. 核心资源** | 用 Trae 写好的课表解析 React 页面；部署在本地的“龙虾（OpenClaw）”助理负责定时执行脚本。 | 依靠 AI 编程和本地智能体保障极低维护成本。 |
| **7. 重要合作** | 阿里云提供免费额度云服务器；大模型 API 服务商提供接口；班长提供各专业最新课表表格。 | 整合校园资源与免费云服务。 |
| **8. 成本结构** | 服务器运行费 0 元（免费额度内）；API 算力费 5 元/月；每日课表更新维护时间 10 分钟。 | 极低运营成本（OPEX），近乎无成本运行。 |
| **9. 收入来源** | 基础服务免费；抢课自动短信催促提醒收 5 元/学期。 | 免费获取流量，低门槛附加值服务变现。 |

> **📊 小华的单元经济模型审计**：
> 一个付费用户一学期消费 5 元。API 消耗 Token 成本为 0.2 元。获客成本（打印海报）分摊为 0.5 元/人。
> 
> $$\text{单客净利润} = 5 - 0.2 - 0.5 = 4.3 \text{ 元}$$
> 
> 当小华在学校里拓展了 500 个付费大一新生时，他一学期能净赚 $500 \times 4.3 = 2150$ 元。而他每天需要做的事情，只是让“龙虾（OpenClaw）”在后台跑着脚本，每天花 10 分钟看看服务器日志。小华的“一人公司”闭环完美跑通！

## 四、 本章行动检查清单

恭喜你完成了第三章“觉醒”的学习！这标志着你已经跨越了“技术好手”与“商业个体”的边界。请在今晚入睡前，尝试完成以下动作：

*   [ ] **清单 1**：在一张白纸上画出九个格子，参考小华的案例，画出你的**第一版个人商业模式画布**。
*   [ ] **清单 2**：算一算你的“单元经济模型”：哪怕只帮人润色一篇作文或者做一个简单的课表导入，扣除 API 费后，你能赚到钱吗？
*   [ ] **清单 3**：想出你的 MVP 计划：如何在 3 天内不写代码，先用手动微信代劳的方式拿到你的第一个测试客户？

商业不是终点，商业是手段。当你通过自己的一人公司，在数字世界里赚到第一笔财富，并意识到你可以不需要依附于任何机构也能体面生活时，你的心智才算真正完成了一次觉醒。
